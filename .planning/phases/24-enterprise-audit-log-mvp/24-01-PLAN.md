<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| policy engine -> audit store | Security decisions cross into durable evidence records |
| tool execution output -> audit event payload | Potentially sensitive output is transformed into persisted metadata |
| runtime query surface -> operator output | Audit data leaves storage for human/system consumption |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01 | T | audit records at rest | mitigate | Use append-only writes with `prev_hash` and `event_hash` chain validation for tamper evidence. |
| T-24-02 | I | sensitive tool output logging | mitigate | Persist hashes and redacted previews only; block raw secret-bearing payload persistence. |
| T-24-03 | R | missing denial traceability | mitigate | Emit policy-decision events before execution for allow/approval/deny outcomes with reason codes. |
| T-24-04 | D | query performance degradation | mitigate | Add SQLite indexes on timestamp/path/tool/decision/outcome and bounded query defaults. |
</threat_model>
