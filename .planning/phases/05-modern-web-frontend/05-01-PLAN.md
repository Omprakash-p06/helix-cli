# Phase 5, Plan 1: Modern Web Frontend

## Objective

Deliver the Web Interface requested by REQ-08. This requires building a standalone React application with TailwindCSS and refactoring the Rust orchestrator (`agent-rs`) to function as an HTTP/WebSocket API when booted in Web mode, rather than a terminal REPL.

## Context & Architecture
To ensure the Rust orchestrator retains full control over the agent logic, tool calling, and LLM compaction, the web interface must be a "dumb client". 

When a user selects "Web Interface" in `start.py`, the system will:
1. Pass `HELIX_UI_MODE=web` to Rust.
2. `agent-rs` will boot an `axum` HTTP API server on port 3000 (instead of `rustyline`).
3. `start.py` will boot `npm run dev` in the `web-ui` folder.
4. The web UI will communicate with `agent-rs` via REST to fetch Agent/Chat responses.

## must_haves
- Premium UI with TailwindCSS (dark mode, glassmorphism, fluid animations).
- The Rust orchestrator must support a local `axum` API endpoint (e.g., POST `/chat`) that handles the iterative tool-calling loop and returns the final response.
- Support for Server-Sent Events (SSE) or simple sequential chunking to stream tool execution statuses to the frontend before the final answer arrives.
- Ensure `start.py` gracefully terminates both the web server and the rust server on exit.

<read_first>
- agent-rs/Cargo.toml
- agent-rs/src/main.rs
</read_first>

<action>
1. Add `axum`, `tower-http` (for CORS), and `tokio` multi-threading features to `Cargo.toml`.
2. Refactor `agent-rs/src/main.rs`: 
   - Parse a new `HELIX_UI_MODE` env var (defaulting to `terminal`).
   - Standardize the core tool-loop into a separate async function `process_agent_turn(input: String)`.
   - If `terminal`, run the existing `rustyline` loop.
   - If `web`, boot an `axum` server exposing `POST /chat`. This endpoint accepts the user prompt, invokes the agent loop, and streams progress updates using SSE (Server-Sent Events) ending with the final text output.
</action>

<acceptance_criteria>

</acceptance_criteria>
