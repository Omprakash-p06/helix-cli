# Phase 2, Plan 1: Rich Terminal Input with rustyline

## Objective

Replace the naive `io::stdin().read_line()` input loop in the Rust orchestrator with `rustyline`, enabling:
- Multi-line paste support via bracketed paste mode (built-in since rustyline v8+)
- Multi-line editing (Enter inserts newline; empty line submits)
- Persistent command history across sessions
- Cross-platform compatibility (Windows, Linux, Arch)

## must_haves
- User can paste a 50-line code block and have it arrive intact as a single input
- Enter on a non-empty line inserts a newline (allows multi-line composition)
- Empty line (Enter on blank) or Ctrl+D submits the full multi-line buffer
- Command history persists to `~/.helix_history`
- Existing `--prompt` eval mode remains untouched
- `quit`/`exit` commands still work

### Task 2: Implement a custom Validator for multi-line input

<read_first>
- agent-rs/src/main.rs (lines 158-200)
</read_first>

<action>
Create a new file `agent-rs/src/input.rs` that encapsulates the rustyline editor setup:

```rust
use rustyline::validate::{ValidationContext, ValidationResult, Validator};
use rustyline::{Completer, Helper, Highlighter, Hinter};
use rustyline::Editor;
use rustyline::error::ReadlineError;
use rustyline::DefaultEditor;

/// Multi-line input helper.
/// Rules:
///   - If the current line is empty (user pressed Enter on blank line), submit.
///   - Otherwise, treat Enter as "insert newline" (continue editing).
#[derive(Completer, Helper, Highlighter, Hinter)]
pub struct MultiLineHelper;

impl Validator for MultiLineHelper {
    fn validate(&self, ctx: &mut ValidationContext) -> rustyline::Result<ValidationResult> {
        let input = ctx.input();
        // Submit on empty input or if the last line is blank (double-Enter)
        if input.is_empty() || input.ends_with('\n') {
            Ok(ValidationResult::Valid(None))
        } else {
            Ok(ValidationResult::Incomplete)
        }
    }
}

pub fn create_editor() -> Result<Editor<MultiLineHelper, rustyline::history::FileHistory>, ReadlineError> {
    let config = rustyline::Config::builder()
        .auto_add_history(true)
        .bracketed_paste(true)
        .build();

    let helper = MultiLineHelper;
    let mut rl = Editor::with_config(config)?;
    rl.set_helper(Some(helper));

    // Load history from user home
    let history_path = dirs_history_path();
    let _ = rl.load_history(&history_path);

    Ok(rl)
}

pub fn save_history(rl: &mut Editor<MultiLineHelper, rustyline::history::FileHistory>) {
    let path = dirs_history_path();
    let _ = rl.save_history(&path);
}

fn dirs_history_path() -> std::path::PathBuf {
    if let Some(home) = std::env::var_os("HOME")
        .or_else(|| std::env::var_os("USERPROFILE"))
    {
        std::path::PathBuf::from(home).join(".helix_history")
    } else {
        std::path::PathBuf::from(".helix_history")
    }
}
```

Register the module in `main.rs` with `mod input;`.
</action>

- `input.rs` contains `impl Validator for MultiLineHelper`
- `input.rs` contains `bracketed_paste(true)`

**Before (current):**
```rust
print!("> ");
io::stdout().flush()?;
let mut stdin = String::new();
if io::stdin().read_line(&mut stdin).is_err() {
    continue;
}
let input = stdin.trim().to_string();

    continue;

    break;
}
```

**After (rustyline):**
```rust
// Before the main loop, create the editor:
let mut rl = input::create_editor().expect("Failed to initialize terminal editor");

// Inside the loop:
match rl.readline("> ") {
    Ok(line) => {
        let trimmed = line.trim().to_string();
        if trimmed.is_empty() {
            continue;
        }
        if trimmed.eq_ignore_ascii_case("quit") || trimmed.eq_ignore_ascii_case("exit") {
            input::save_history(&mut rl);
            break;
        }
        trimmed
    }
    Err(ReadlineError::Interrupted) => {
        // Ctrl+C — skip current input
        continue;
    }
    Err(ReadlineError::Eof) => {
        // Ctrl+D — exit
        input::save_history(&mut rl);
        break;
    }
    Err(err) => {
        println!("[Input Error] {}", err);
        continue;
    }
}
```

Also add `use rustyline::error::ReadlineError;` at the top of main.rs.

Keep `--prompt` eval mode completely untouched — it bypasses the interactive loop.

At the end of the main function (after the outer `loop` breaks), add `input::save_history(&mut rl);` to persist history on clean exit.
