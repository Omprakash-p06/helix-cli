<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| terminal user -> `TuiAction` | Untrusted key input controls layout changes, sidebar toggles, and command dispatch. |
| orchestrator -> `TuiEvent` | Model/tool text crosses into the renderer and must not break layout state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | T | `agent-rs/src/tui/api.rs` contract | mitigate | Define one shared enum/struct surface and remove duplicate ad hoc event shapes so actions/events cannot silently drift between `main.rs` and `tui.rs`. |
| T-19-02 | D | `agent-rs/src/tui.rs` shell renderer | mitigate | Clamp layout mode to `wide` or `compact`, keep sidebar width bounded, and preserve existing streaming state updates so malformed layout input cannot panic or blank the UI. |
| T-19-03 | I | status/session HUD | mitigate | Render only explicit `ContextSnapshot` metadata from the orchestrator and keep connection/model strings structured instead of formatting raw backend payload blobs directly into the shell. |
</threat_model>
