# Plan 25-01: Tool Registry & IDE API Foundation

This plan refactors the hardcoded tool dispatching into a registry system and exposes discovery endpoints for IDE integration.

## Tasks

### 1. Tool Registry Implementation (ENT-02)
- Define `Tool` trait in `agent-rs/src/tools.rs` with `name()`, `description()`, `schema()`, and `execute()` methods.
- Implement `ToolRegistry` struct to manage a collection of `Box<dyn Tool>`.
- Wrap existing tools (read_file, run_command, etc.) into `Tool` trait implementations.

### 2. Orchestrator Refactoring
- Update `execute_tool_sync` in `agent-rs/src/main.rs` to use the `ToolRegistry` instead of `match ToolCallArgs`.
- Initialize the registry in `main()` and pass it to execution loops.

### 3. IDE Bridge API (IDE-01)
- Add `GET /v1/status` to `agent-rs/src/server.rs` returning model health and orchestrator version.
- Add `GET /v1/tools` to `agent-rs/src/server.rs` returning the registered tool schemas for IDE discovery.
- Implement a simple "System Context" API `GET /v1/context` that returns active workspace root and git branch.

## Success Criteria
- All 7 built-in tools are successfully dispatched via the `ToolRegistry`.
- `curl http://localhost:3000/v1/tools` returns valid JSON schemas for all tools.
- `cargo test` passes with the new registry-based dispatch logic.
