# Plan 06-01: GPU Process Cleanup & iGPU Fallback

## Objective
Implement aggressive resource cleanup in `start.py` and `scripts/start_server.py` to prevent VRAM accumulation, and add retry logic to fallback to iGPU (or system RAM) when `llama-server` fails due to Out-Of-Memory.

## Requirements Addressed
- PERF-01: The launcher must automatically clear lingering GPU memory (e.g., orphaned `llama-server` or `koboldcpp` processes) before booting a new agent.
- PERF-03: The system must implement a fallback mechanism to route inference to the iGPU if the primary dGPU VRAM is exhausted.

## Tasks

<task>
<read_first>
- start.py
</read_first>
<action>
In `start.py`, before launching the background `scripts/start_server.py` process, implement a function `clean_orphaned_servers()`.
This function should use `psutil` (if available) or native OS commands (`taskkill /F /IM llama-server.exe` on Windows, `pkill -f llama-server` on Linux) to terminate any pre-existing `llama-server.exe`, `llama-server`, `koboldcpp.exe`, or `koboldcpp` processes.
This guarantees the VRAM is flushed before allocating the new model.
</action>
<acceptance_criteria>
`start.py` contains the execution of process kill commands for `llama-server` and `koboldcpp`.
</acceptance_criteria>
</task>

<task>
<read_first>
- scripts/start_server.py
- scripts/config.py
</read_first>
<action>
In `scripts/start_server.py`, modify the `subprocess` call for `llama-server` to allow it to fail fast. If the backend process crashes immediately (e.g. exit code 1) and writes "out of memory" or "failed to allocate" to stderr, the script must catch this.
When Caught: Automatically retry launching the server but override the GPU layers to point to the integrated GPU or system RAM (e.g., setting `-ngl 0` or dropping the Vulkan device index). 
Update `scripts/config.py` to include `FALLBACK_GPU_LAYERS = 0`.
</action>
<acceptance_criteria>
`scripts/start_server.py` contains a retry loop that checks `.poll()` and reads stderr for OOM messages, subsequently falling back to CPU/iGPU.
</acceptance_criteria>
</task>

## Verification
1. Start a fake `llama-server.exe` process. Run `start.py` and verify it gets killed.
2. Force an OOM state manually by starting `start_server.py` with `GPU_LAYERS=999`. Verify it catches the failure and retries with `FALLBACK_GPU_LAYERS`.
