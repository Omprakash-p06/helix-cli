# Plan: GSD Workflow Integration

## Objective
Integrate GSD (Get Shit Done) commands for structured project management: /gsd new-project, /gsd discuss-phase, /gsd plan-phase, /gsd execute-phase, /gsd verify-work.

## Tasks

### Task 1: Create GSD Directory Structure
- [ ] Create `~/.config/helix-agent/gsd/` directories
- [ ] Copy GSD prompts/templates

### Task 2: Add GSD Commands
- [ ] Add /gsd-* commands to command palette
- [ ] Route to gsd_handler module

### Task 3: Implement map-codebase
- [ ] Recursively scan directory
- [ ] Generate project map markdown

### Task 4: Implement new-project
- [ ] Interactive questioning workflow
- [ ] Generate research, requirements, roadmap

### Task 5: Implement discuss-phase
- [ ] Read existing plan
- [ ] Enter discussion mode
- [ ] Save decisions

### Task 6: Implement plan-phase
- [ ] Generate detailed plan with tasks
- [ ] Include success criteria

### Task 7: Implement execute-phase
- [ ] Execute tasks sequentially
- [ ] Verify after each task
- [ ] Commit on success

### Task 8: Implement verify-work
- [ ] Run UAT tests
- [ ] Produce report

## Success Criteria
- [ ] /gsd new-project creates .gsd/ directory
- [ ] /gsd discuss-phase captures decisions
- [ ] /gsd plan-phase generates actionable plan
- [ ] /gsd execute-phase runs and commits tasks
- [ ] /gsd verify-work runs UAT and reports

## Files
- `agent-rs/src/commands/gsd.rs` - GSD command handlers
- `agent-rs/src/gsd/prompts.rs` - GSD system prompts
- `agent-rs/src/gsd/executor.rs` - Phase execution logic
