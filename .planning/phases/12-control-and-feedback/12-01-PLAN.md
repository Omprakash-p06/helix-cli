# Plan 01: Interactive Controls and Telemetry

This plan adds granular UI controls and feedback loops to the ratatui interface to let users abort generations comfortably, navigate past message history seamlessly, and gauge local model performance natively.

## Task 1: Interrupt Handler and TTFT State Updates (agent-rs/src/tui.rs)
Extend the existing TUI definitions and update input handling to support cancellation and live TTFT tracking.

**Steps:**
1. **Enums & Structs**:
   - Extend `TuiAction` with `Interrupt`.
   - Add new `TuiEvent` variants: `StatusUpdate(String)` (to push detailed string updates) and `GenerationStarted` (to signal TTFT lock).
   - In `TuiApp`, add tracking variables: `ttft_start: Option<Instant>` and `ttft_locked: Option<Duration>`.
2. **Key Handler Updates**:
   - In `handle_key_event`, check if `app.is_generating` is true when `Ctrl+C` is pressed. If true, emit `TuiAction::Interrupt` and block exiting the app. Only emit `Quit` if `is_generating` is false.
   - For TTFT, `app.ttft_start` is initialized when the orchestrator sets `is_generating = true` (or via an explicit orchestrator signal).
3. **Status Bar Updates**:
   - If `ttft_start` is active and `ttft_locked` is none, dynamically update the status bar to show `Thinking... ({elapsed}s)` instead of static `Generating...`.
   - If `ttft_locked` is set, append `[TTFT: {duration}s]` to `status_text`.

## Task 2: Orchestrator Mid-stream Interruption and TTFT Signal (agent-rs/src/main.rs)
Modify the asynchronous `main.rs` streaming loop to catch `Interrupt` signals and cleanly break out of the generation step while saving accumulated tokens.

**Steps:**
1. In `run_llm_loop_tui`:
   - Start an `Instant::now()` timer when the POST request to the inference server begins.
   - Send periodic `TuiEvent::StatusUpdate` ticks (every 100ms) with `Thinking... ({elapsed}s)` via the existing `flush_interval` (which we can increase its frequency during wait, but let's stick to simple duration pushing, or better let the `tui.rs` handle the tick locally to avoid channel spam). Let's emit a precise `GenerationStarted` event upon the arrival of the *very first* `TokenChunk`.
2. **Handle Interrupt `tokio::select!`**:
   - Inside the polling loop spanning `stream.next()` and `flush_interval.tick()`, add a branch to check `action_rx` (if we split rx handling, or safely read from a shared interrupt flag/channel).
   - Since `action_rx` is managed centrally in `main`, we can read it directly inside the `tokio::select!` loop while generating.
   - If an `Interrupt` action is received, immediately call `break;`, push the remaining `token_buffer`, finalize the generation, and push a localized system span `[Generation Interrupted]`.

## Task 3: Chat History Scrolling (agent-rs/src/tui.rs)
Implement full page up/down buffer navigation independent of the input prompt.

**Steps:**
1. **Input Key Checks**:
   - Identify `PageUp` and `PageDown` in `handle_key_event`. Increase/decrease `app.scroll_offset` by the term height (or 5 lines).
   - For `Up`/`Down`, read the `multiline_buffer` and `app.input` line length. If completely zero/empty, hijack `Up`/`Down` logic and adjust `scroll_offset` by 1 line incrementally.
2. **Scroll Clamping**:
   - Validate the maximum safe scroll bound in `draw_chat_area` based on total rendered line count against active terminal layout constraints.
   - Ensure the user bounds correctly to 0 (bottom of screen/auto-scroll mode).
3. **Render Adjustment**:
   - Slice or step backwards through the flattened render `lines` vector so that `draw_chat_area` renders the appropriate slice offset from the bottom of the buffer.
