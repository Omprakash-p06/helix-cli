<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user shell -> installer CLI | Untrusted model references and flags enter the install flow here |
| installer -> Hugging Face Hub | Remote metadata and model artifacts are fetched from a third-party service |
| staging path -> active models directory | Downloaded artifacts cross into executable runtime state here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-01 | T/S | shared model resolver | mitigate | Pin revision and repo identity in the trusted registry; reject any model reference that does not match the allowlist entry exactly. |
| T-21-02 | T | download and activation flow | mitigate | Download to a staging path, compute SHA-256 locally, and fail closed if the digest does not match the expected allowlist value before moving the file into `models/`. |
| T-21-03 | E/T | filename handling | mitigate | Normalize remote filenames to basename-only active paths and refuse any path segments that would escape the models directory or overwrite unrelated files. |
</threat_model>
