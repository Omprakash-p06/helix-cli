# Phase 1, Plan 1: Boot Upgrades, Mode Selection & Persona Cleanup

## Objective

Upgrade `start.py` to present the user with a structured boot menu for selecting UI mode (Terminal/Web) and execution mode (Agentic/Chat), then pass those choices into the Rust orchestrator. Also enforce cross-platform parity (Windows/Linux/Arch) and strip all LLM personality/hallucination trails from the system prompts.

## must_haves
- `start.py` presents a clear 2-step mode selection menu before launching any backend
- User selection of Terminal vs Web and Agentic vs Chat is captured and forwarded
- Rust orchestrator receives mode as an env var and adjusts behavior (tools enabled/disabled)
- System prompts contain explicit anti-hallucination and anti-greeting directives
- Cross-platform compatibility verified (no Windows-only paths or commands assumed)

## Tasks

### Task 1: Refactor `start.py` boot menu

<read_first>
- start.py
- scripts/config.py
- scripts/helix_branding.py
</read_first>

<action>
Replace the current linear boot flow in `start.py` with a structured 2-step menu after model selection:

**Step 1 — Interface Selection:**
```
Select interface:
  1) Terminal Chat
  2) Web Interface (coming soon)
```

**Step 2 — Execution Mode Selection:**
```
Select mode:
  1) Agentic (autonomous tool-calling)
  2) Chat (conversation only)
```

Capture these as variables `interface_choice` and `exec_mode`. Pass `exec_mode` into the Rust orchestrator via `server_env["HELIX_EXEC_MODE"]` (value: `"agentic"` or `"chat"`).

If interface_choice == "2" (Web), print `"Web interface is planned for Phase 5. Falling back to Terminal."` and continue with Terminal.

Remove the hardcoded `print("Agent mode:   default")` line. Replace with actual mode display:
```python
print(f"Interface: Terminal")
print(f"Mode:      {exec_mode.capitalize()}")
```

Use `input()` for prompts — this is cross-platform and works identically on Windows, Linux, and Arch.
</action>

<acceptance_criteria>
- `start.py` contains `HELIX_EXEC_MODE` assignment to `server_env`
- `start.py` contains `Select interface:` prompt string
- `start.py` contains `Select mode:` prompt string
- `start.py` does NOT contain `Agent mode:   default`
</acceptance_criteria>

### Task 2: Rust orchestrator reads exec mode and adjusts behavior

<read_first>
- agent-rs/src/main.rs
- agent-rs/src/config.rs
</read_first>

<action>
**In `config.rs`:**
Add a new field to `AppConfig`:
```rust
pub exec_mode: String, // "agentic" or "chat"
```

In `load_from_python()`, add to the Python bridge script:
```python
"exec_mode": os.environ.get("HELIX_EXEC_MODE", "agentic"),
```

**In `main.rs`:**
Read `app_config.exec_mode`. If `exec_mode == "chat"`:
- Set `tools_payload` to `json!([])` (empty tools array — pure chat, no tool definitions sent)
- Skip critic injections entirely

If `exec_mode == "agentic"`:
- Behavior identical to current (tools enabled, critic active)
</action>

### Task 4: Cross-platform parity audit of Python scripts

<read_first>
- start.py
- scripts/start_server.py
- scripts/config.py
</read_first>

<action>
Audit and fix all cross-platform issues:

1. **`scripts/config.py`:** The `KOBOLD_BIN` is hardcoded to `"koboldcpp.exe"`. Change to:
   ```python
   import platform
   KOBOLD_BIN = "koboldcpp.exe" if platform.system() == "Windows" else "koboldcpp-linux-x64"
   ```
   Note: `setup.py` already handles platform-specific downloads, but the config must match.

2. **`start.py`:** The `os.system('cls' if os.name == 'nt' else 'clear')` line is correct — keep it.

3. **`scripts/start_server.py`:** Line 13 has `from helix_branding import print_helix_logo` which relies on `sys.path.insert(0, SCRIPT_DIR)` at line 19. This import ordering only works if Python is invoked from the scripts directory. Reorder so that `sys.path.insert` happens BEFORE the import:
   ```python
   SCRIPT_DIR = os.path.dirname(os.path.abspath(__file__))
   PROJECT_DIR = os.path.dirname(SCRIPT_DIR)
   sys.path.insert(0, SCRIPT_DIR)
   from helix_branding import print_helix_logo
   ```

4. **`scripts/config.py`:** `DANGEROUS_COMMANDS` contains Linux-only commands (`chmod`, `dd`, `mkfs`, `fdisk`, `systemctl`). Add Windows equivalents:
   ```python
   if platform.system() == "Windows":
       DANGEROUS_COMMANDS = ["del", "rmdir", "format", "diskpart", "shutdown", "rm", "mv"]
   else:
       DANGEROUS_COMMANDS = ["rm", "mv", "chmod", "dd", "mkfs", "fdisk", "systemctl", "reboot", "shutdown"]
   ```
</action>

<acceptance_criteria>
- `scripts/config.py` contains `platform.system()` check for `KOBOLD_BIN`
- `scripts/config.py` contains `platform.system()` check for `DANGEROUS_COMMANDS`
- `scripts/start_server.py` has `sys.path.insert(0, SCRIPT_DIR)` BEFORE `from helix_branding import print_helix_logo`
- `start.py` does NOT contain any Windows-only hardcoded paths
</acceptance_criteria>
