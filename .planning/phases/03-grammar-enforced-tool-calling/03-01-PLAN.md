# Phase 3, Plan 1: Grammar-Enforced Tool Calling

## Objective

Guarantee 100% deterministic and syntactically correct tool calls by dynamically generating a GBNF (GGML BNF) grammar from the active tools' JSON schemas, and enforcing it at the inference engine level.

## must_haves
- Automatic extraction of JSON tool schemas from Rust types (already handled by `schemars`).
- Dynamic conversion of the tool schemas into a valid GBNF string (using the `gbnf` Rust crate).
- Injection of the `grammar` parameter into the API request payload securely.
- The grammar must allow the model to natively choose between making a valid tool call JSON block OR generating conversational text.
- Fallback/graceful handling if the inference backend rejects the grammar.

### Task 2: Implement dynamic GBNF converter in `tools.rs`

<read_first>
- agent-rs/src/tools.rs
</read_first>

<action>
1. In `tools.rs`, create a new function `pub fn generate_tool_grammar(tools_payload: &serde_json::Value) -> String`.
2. This function will reconstruct an overall JSON schema that wraps the available tools. 
Alternatively, since the `gbnf` crate accepts a full JSON Schema, we construct a compound schema representing the LLM's valid output format:
```json
{
  "oneOf": [
    { "type": "string" }, 
    { "type": "object", "properties": { "tool_calls": { ... tools schema ... } } }
  ]
}
```
3. Use the `gbnf` crate to convert this dynamically constructed `serde_json::Value` representing the grammar boundary schema into a `String` (GBNF format).
</action>

<acceptance_criteria>
- `tools.rs` exports `generate_tool_grammar`
- Function uses the `gbnf` crate to compile the JSON schema tree to GBNF
- Returns a raw GBNF string
</acceptance_criteria>

---

### Task 3: Inject `grammar` into API payload in `main.rs`

<read_first>
- agent-rs/src/main.rs (around line 287)
</read_first>

<action>
1. During initialization in `main.rs` (where `tools_payload` is generated), also call `generate_tool_grammar(&tools_payload)`. Keep the generated grammar in memory.
2. If `exec_mode` is `chat`, skip grammar injection.
3. If `exec_mode` is `agentic`, inject the grammar into the `request_body`:
```rust
let mut request_body = json!({
    "model": app_config.model_name,
    "messages": &messages,
    "tools": tools_payload,
    "temperature": temperature_override
});

if !is_chat_mode {
    request_body.as_object_mut().unwrap().insert(
        "grammar".to_string(),
        json!(generated_grammar_string)
    );
}
```
4. Handle any runtime API rejections gracefully (if KoboldCpp/llama.cpp complain about grammar format, log a warning and fall back to skipping the grammar field for that round).
</action>

<acceptance_criteria>
- `main.rs` contains grammar generation call.
- `main.rs` injects `"grammar"` field conditionally.
- If inference engine fails with HTTP 400 (Bad Request) on the custom grammar, Rust catches it and logs gracefully.
</acceptance_criteria>
