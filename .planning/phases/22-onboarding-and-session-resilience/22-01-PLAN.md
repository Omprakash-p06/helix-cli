<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input -> slash command parser | Untrusted command text and session names enter persistence operations here |
| runtime memory -> session files | In-memory conversation state crosses into durable disk artifacts |
| startup launcher -> runtime resume selection | Resume intent and profile defaults cross process boundaries |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01 | T | session file writer | mitigate | Use atomic temp-write + rename and validate serialized envelope fields before marking save successful. |
| T-22-02 | D | startup resume loader | mitigate | On malformed/corrupt session payload, fail safe to fresh session with warning instead of panic or endless retry loop. |
| T-22-03 | E/T | slash command name handling | mitigate | Normalize and sanitize session names (basename-only, deny traversal tokens) before file operations. |
| T-22-04 | R | lifecycle command operations | mitigate | Emit deterministic system messages and timestamped metadata so save/load/resume outcomes are auditable during troubleshooting. |
</threat_model>
