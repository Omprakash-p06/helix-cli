# Phase 4, Plan 1: KoboldCPP Fallback Accuracy

## Objective

Formally verify and ensure that the backend fallback engine, KoboldCPP, achieves identical tool-calling accuracy to the primary `llama.cpp` server. 

## Context & Architecture
During Phase 3, GBNF grammar generation was implemented globally. The raw `grammar` parameter is injected into the root of the OpenAI `/v1/chat/completions` payload. Because KoboldCPP is a highly compliant fork of `llama.cpp` regarding API payload structural parsing, it natively inherits the Phase 3 GBNF enforcement without requiring a separate bridging layer.

The only difference is that KoboldCPP requires `strict: true` flags to be stripped from individual tool schemas to avoid 400 Bad Request errors on older versions. This is already handled natively by `strict_tools = false`.

## must_haves
- Update the runtime boot logs in Rust to explicitly reflect that KoboldCPP is inheriting global GBNF enforcement.
- Verify that the `server_flavor` detection works and correctly applies both the stripped tool schema AND the strict GBNF grammar.
